Create a Personal Website With Hugo

December 27, 2022 in Hugo, post

Introduction

I thought to build a personal website several times, but failed to do it. At one weekend of December, 2022 I decided to complete this website project before the new year 2023. I started by searching “create a personal website” in youtube. The algorithm recommends videos with Hugo. My first thought ðŸ¤”: since when Hugo Boss started to build websites instead of selling expensive clothes ðŸ˜„. Some videos promise to do it with Hugo within 10 minutes. That’s great. I was excited about it.

The main reasons I want to have a personal website are:

  • writing blogs: I continue learning new skills and technologies. I want to summarize and document them down by writing blogs.
  • digital portfolie: showcase my past projects and share knowledge.

Hugo is a static site generator which allows website authors to create blogs or pages in Markdown. So it’s suitable for me. You can host it freely in Github pages. What’s more, there are hundreds of Hugo themes ready to use for various purpose. Sounds cool? And guess what? I spent 10 minutes to watch that youtube turorial but the rest of weekend to figure it what’s wrong with one error by deploying. I will share my guide so that you don’t have to go through all the pain I went throught.

Prerequisites

Hugo is pretty cool and seems easier, you are expected to already know some basic knowledge of html, command line, git etc. It requires investing time to learn some things if you don’t know them all.

  • Installation of Hugo
    Refer to the official documentation Installation
  • Basic knowledge of Markdown or HTML, css, command line and text editors
    Both HTML and Markdown are supported content formats.
  • One IDE, e.g. VS code
  • A Github account
    https://pages.github.com/
  • Knowledge of Git
    Git is required to install a theme as a Git submodule, access commit information from a local repository and host your site.

Choose a Hugo Theme

You can find one suitable Hugo theme hier. Either for a portfolio, or company, there are plenty of template themes to choose. I’ve chosen hugo tranquilpeak theme.

Create a project

  1. Create a folder in your local PC. If you want to use any hugo theme, you can do it either with git clone or git submodule. git submodule makes it easier to update this theme if you have your Hugo site in git as well.

git submodule is recommended. You can always copy the layouts to overwrite the default theme.

mkdir hugo-site
hugo new site my-hugo-site
cd my-hugo-site
git init
## Add theme
git submodule add https://github.com/kakawait/hugo-tranquilpeak-theme.git themes/hugo-tranquilpeak-theme
echo "theme = 'hugo-tranquilpeak-theme'" >> config.toml
hugo server

git clone the theme under the folder themes. Please don’t forget to remove .git.

Create your first content

hugo new posts/my-first-blog.md

You can use markdown language to write content.

Deploy in Github

I tried two methods to deploy in Github.
The first one with public is recommended. I wrote down the second method for reference.

  1. Method:
    With Hugo command you can publish your website. then enter Public folder and git init. Push this public folder to a github repository.
    In this method, you will prepare two github repositories. One private repo is for the content and the public repo is for the publishing website.

  2. Method:
    In the root folder create the folders called .github/workflows. Then create a file called gh-pages.yml. Then copy the following content to the yml file. Then git push all the changes in the root folder to github repository.

name: github pages

on:
  push:
    branches:
      - main  # Set a branch that will trigger a deployment
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive)
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          # extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        if: github.ref == 'refs/heads/main'
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
